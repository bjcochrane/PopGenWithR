Background

As we’ve seen, in order to interpret structure runs, we need to work with multiple runs at a given k. STRUCTURE itself can readily visualize single runs, Structure Harvester is an easy online way to identify what value of K is worth examining, but what we would like to do is, having identified a K of interest, combine and visualize multiple runs. There is a commonly used pipeline for doing so, involving the use of two command-line programs

  1. [CLUMPP] creates combined files
  2. [distruct] produces graphical output

The problem is that both require some finesse in manually modifying parameter files, and many of those modifications are tricky. As an alternative, the workflow below allows you to achieve the same goal using the R package pophelper. It is available in github, so we have the usual initial installation steps:

# library(devtools)
# install_github("royfrancis/pophelper")
library(pophelper)

The Evanno method

We can execute the Structure Harvester algorithm quite readily within R, and we can create either a simple plot or have our package generate a more elegant .png file. To just get the deltaK vs K plot, we can do the following:

df.ev <-evannoMethodStructure(df2)
plot(df.ev$k,df.ev$deltaK,xlab="K",ylab="deltaK",type="l")

Or, to generate a more elaborate file, we can run the same code as

df.ev <-evannoMethodStructure(df2,exportplot = TRUE)
## evannoMethodStructure.png exported

this has now created a file called evannoMethodStructure.png, which can be opened with any graphical program, and looks like the following

Combining Runs with CLUMPP

There are actually a couple of problems with raw STRUCTURE output. First, in each run, the ordering of clusters (1 through K) is arbitrary, so that even though different runs may cluster samples identically (or nearly so), the numbering in each run is not the same (for example, cluster 1 in run 1 might be cluster 4 in run 2). Second, for display purposes, assuming that the runs have proven to be similar, we’d like to merge them into a single plot. CLUMPP will do both of those things.

1. Produdcing CLUMPP input files.

CLUMPP requires two input files, one containing the combined results for a particular K and one with parameters for running the program. We can create these for each of our K values quite readily with the following

2. Running CLUMPP

To run CLUMPP, we need to have the CLUMPP executable file in the same directory as our two input files (those in Caribou_K6) We’ll do this in R, but if you prefer, you can do so just by moving and double clicking. First the R version:

file.copy("CLUMPP","./Caribou_K6/CLUMPP") # Make a copy of CLUMPP in the K6 file
## [1] TRUE
setwd("./Caribou_K6") # Move our R processing to that directory
system("./CLUMPP") # Run CLUMPP
list.files() # display the files in the folder # List the files
## [1] "Caribou_K6-combined-aligned.txt"  "Caribou_K6-combined-merged.txt"  
## [3] "Caribou_K6-combined-miscfile.txt" "Caribou_K6-combined.txt"         
## [5] "CLUMPP"                           "paramfile"
setwd("..")

We now see a bunch of new stuff in the folder, much of which we can ignore. However, we will use one of those files to visualize the results.

As an alternative (on a Mac), you can do the following.

  1. Open Finder
  2. Navigate to the folder that contains the CLUMPP executable
  3. Copy the executable into the folder with the input files (Caribou_K6 in the example)
  4. With that folder open in Finder, double click on CLUMPP. You should see the additional files appear

3. Visualizing the results.

This can be done with the program distruct, but as noted earlier, setting that up is a bit tricky. Fortunately for us, pophelper can do that fairly easily. First, let’s look at our three runs as they were produced by STRUCTURE

plotRuns("./Caribou_K6/Caribou_K6-combined.txt",imgoutput = "tab")
## Caribou_K6-combined.png exported

The .png file produced shows the problem - the clusters are clearly the same, but the coloring (indicating cluster number) differs between runs.

If we look at the CLUMPP output, we see that this problem is eliminated

plotRuns("./Caribou_K6/Caribou_K6-combined-aligned.txt",imgoutput = "tab")
## Caribou_K6-combined-aligned.png exported

Since these are clearly consistent, we can merge them:

plotRuns("./Caribou_K6/Caribou_K6-combined-merged.txt",imgoutput = "tab")
## Caribou_K6-combined-merged.png exported

And again, a .png file is created.

Note that at present, this is just a basic command; in fact the plotRuns function has a lot of options for labelling everything. Stay tuned for more.
